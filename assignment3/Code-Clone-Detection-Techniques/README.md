### Environment Setup
We used firefox verison 148.0.2 and prompted using prompts


We opened a fresh chat in a private browser over two sessions to ensure no prior memories affected results. A preliminary training prompt was provided, similar to the paper's technique.
This prompt was generated by ChatGPT, though edited down to reduce length as the paper discusses shorter prompts produced better results.

We used ChatGPT to modify the Starchat script to generate 50 pairs of code over two sessions from the big clone bench dataset, with 20 being clones and 30 being not clones. By selecting a fairly consistent mix of YES and NO, we can ensure we catch if the model leans toward one answer if it's unsure.

Below is the prelimenary training prompt we used.

```
You are a software engineering expert specializing in code clone detection.

Two Java functions are given below.

Your task is to determine whether they are code clones.

Clone definitions:

Type-1: Identical code except for whitespace or comments  
Type-2: Same structure but variable or identifier names changed  
Type-3: Mostly similar but with small additions, deletions, or modifications  
Type-4: Different implementation but same functionality  

If the functions implement the same behavior, they are clones.

Respond using ONLY one of the following:

Clone: YES
Type: Type1 / Type2 / Type3 / Type4

or

Clone: NO

Example 1

Function A:
int sum(int a,int b){ return a+b; }

Clone: YES
Type: Type2

int i=0;

 total+=a[i];
 i++;
}

Answer:
Clone: YES
Type: Type4
```

### Results Assessment

ChatGPT classified 38 of 50 pairs correctly, with 3 false positives and 8 false negatives. This is an unimpressive result, as it clearly prefers to answer NO if it's unsure.

It is likely that our pretraining prompt is inferior to the paper's. It is worth noting that the researtchers spent some amount of money finetunning and customizing the prompt on code pairs for it to perform better and we used the free available model rather than fine tunning one using [OpenAI fine training features](https://developers.openai.com/api/docs/guides/supervised-fine-tuning) due to budgetary constraints. There is also a cahance that the online model has changed in some substantial way to make it worse at clone detection. As the paper did not specify their exact training prompt, we had to take an educated guess with our own.

It is worth noting the first session of prompts with ChatGPT was much better than the second, with the 20 pairs classified with only a single mistake. As the methodology was the exact same, it shows that our training prompt may be too inconsistent to rely on long-term.

A pre-trained model is likely the better approach, as you would expect much more consistent results, with the ability to tune more closely without guessing what's happening behind the scenes.

Finally, it is useful to mention that these results are not comparable to the paper's findings. Without their exact prompts, model, and API access, we can only approximate their techniques. We expect with the paper's methods, we would find much better results.
